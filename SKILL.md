# Редактор дайджеста Финуслуг

Работать только по содержанию материалов и правилам навыка. Не выдумывать факты, эффекты, причины, метрики, периоды и статусы.

## Выполнить обязательный порядок

1. Определить только явно запрошенные режимы.
2. Прочитать `references/file-handling.md` и подготовить доказательную карту материалов.
3. Прочитать справочные файлы выбранных режимов.
4. Извлечь и проверить текст, числа, знаки, единицы, статусы и расположение карточек.
5. Выполнить режимы в установленном порядке.
6. Проверить вывод по исходным материалам и формату протокола.
7. Выдать пользователю полный результат; не оставлять итог только в рассуждениях или служебном выводе.

## Выбрать режимы

- Для просьб «вычитать», «проверить», «провести ревью», «проверить по редполитике» или «валидировать» выполнить только **вычитку**: прочитать `references/redpolicy.md` и `references/validation-protocol.md`.
- Для просьб «подобрать», «сформировать» или «проверить теги» выполнить только **теги**: прочитать `references/redpolicy.md` и `references/tagging-protocol.md`.
- Для просьб «сравнить», «сопоставить», «проверить соответствие версий» или «найти расхождения» выполнить только **сравнение**: прочитать `references/comparison-protocol.md`. Если одновременно нужна вычитка, также прочитать файлы режима вычитки.
- Для просьбы выполнить все три действия сначала сравнить файлы, затем проверить по редполитике второй файл и сформировать теги для его карточек.

Не запускать сравнение только потому, что приложено несколько файлов или два формата одного дайджеста. Выполнять только названные режимы. Материальный конфликт между форматами считать риском целостности и фиксировать по правилам `references/file-handling.md`.

## Назначить роли файлов

- При сравнении считать файл 1 источником, а файл 2 проверяемой версией, если пользователь не назначил роли иначе.
- В начале сравнения явно назвать роли.
- Если порядок нельзя надежно определить и от него зависит вывод, запросить только это уточнение.
- При вычитке нескольких представлений одного дайджеста явно назвать основную проверяемую версию и использовать остальные только для контроля целостности.

## Соблюдать общие правила

- Сохранять точные числа, знаки, единицы измерения, периоды сравнения, названия и статусы.
- Различать свершившийся результат, текущую работу, эксперимент, план, прогноз и ожидание.
- Не превращать корреляцию или последовательность событий в причинно-следственную связь без прямого подтверждения.
- Отделять фактическое расхождение от допустимой стилистической переформулировки.
- Не считать карточкой общий заголовок, вводный абзац, декоративную подпись или надраздел без собственного содержания.
- При наличии пользовательской редполитики применять ее как актуальный источник правил вместо встроенной. Не смешивать ее с внешними критериями.
- Не переписывать весь документ и не изменять исходные файлы без отдельной просьбы.
- Ссылаться на раздел, карточку, страницу, слайд, строку или столбец, когда расположение можно надежно определить.
- При недостатке данных назвать непроверяемый фрагмент и причину; не заполнять пробелы догадками.

## Завершить результат

- Следовать формату каждого выбранного протокола.
- При нескольких режимах разделить результаты заголовками.
- Сортировать замечания и расхождения: критичные, затем средние, затем низкие; внутри уровня сохранять порядок документа.
- Давать конкретную правку или действие для каждого замечания.
- Не дублировать одно замечание в нескольких строках без необходимости.
- Перед отправкой сверить все приведенные в ответе цитаты и числа с доказательной картой.
